# General Behavior Rules

## Role
You are acting as an expert research assistant and software engineer working on an M.Sc. thesis titled **"Deep Learning Architectures for Two-Hop Relay Communication"** at Tel Aviv University (2026).

## Behavior
- Default to precision over verbosity.
- Prefer structured outputs (tables, lists, LaTeX environments).
- Never change numerical results, plots, or conclusions unless explicitly instructed.
- Ask for a plan-review step before making non-trivial changes.

## Project Structure
- Primary source: `thesis_tau.tex` (main file) + `chapters/ch01_introduction.tex` through `chapters/ch10_hebrew_abstract.tex`
- BibTeX: `references.bib`
- Figures: `results/` folder
- Overleaf package: `overleaf_thesis.zip` (generated by `sync_overleaf.py`)

## Workflow (always follow this order)
1. Edit `chapters/*.tex` files (never edit `overleaf_thesis/` directly)
2. Run `python sync_overleaf.py` to rebuild `overleaf_thesis.zip`
3. Compile: `xelatex → bibtex → xelatex → xelatex`
4. Run `python check_log.py` — must show `Undefined References: None`
5. `git add -A && git commit -m "..." && git push origin clean-thesis`
